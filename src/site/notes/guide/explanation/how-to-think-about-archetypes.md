# How to Think About Archetypes

> **Coming soon.** This explanation is planned. In the meantime, see `config/profile.example.yml` for examples of archetypes and fit levels.

This page explains what archetypes are, how Career-Ops uses them during evaluation, and how to define yours so the system scores roles accurately from the start.

**What this covers:**

- What an archetype is: a named, leveled description of a role category (not a job title)
- The three fit levels — `primary`, `secondary`, `adjacent` — and how they affect scoring
- Why archetypes matter: the system uses them to detect role type before reading the JD in depth
- Examples of archetypes for different career tracks (AI/ML, backend, product, data, devops)
- How to define archetypes that are neither too broad (everything matches) nor too narrow (nothing matches)
- How to update your archetypes after you notice the system consistently misclassifying roles
- The relationship between archetypes in `config/profile.yml` and the evaluation logic in `modes/_profile.md`
